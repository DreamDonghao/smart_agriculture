# 智慧农业系统（Smart Agriculture）

## 一、项目简介

- 本项目是一个「智慧农业前后端」解决方案，旨在为农业环境提供自动化监控与管理。
- 后端使用 **C++** 开发，前端使用 **Vue 3** 框架。
- 支持传感器数据采集、实时监控、可视化展示等功能。
- 适合用于农田温室监控、灌溉管理、环境调节等智慧农业场景。

## 二、系统架构

1. **后端 (Backend)**
   - 语言：C++ ([GitHub](https://github.com/DreamDonghao/smart_agriculture))
   - 负责接收来自传感器（或模拟传感器设备）的数据，进行处理、存储、分析。
   - 可能包括数据接口 (API)、业务逻辑 (如告警、历史记录)、数据库等模块（视具体实现情况而定）。
2. **前端 (Frontend)**
   - 框架：Vue 3 ([GitHub](https://github.com/DreamDonghao/smart_agriculture))
   - 提供用户界面，用于显示传感器数据（温度、湿度、光照等），展示图表、状态面板。
   - 支持用户交互，例如设置阈值、开关告警、查看历史数据。

## 三、主要功能 (建议模块)

以下是该智慧农业系统可以具有或已经具有的关键功能模块（你可以根据实际代码补充 / 删除）：

- **实时数据采集**：从传感器获取农业环境数据。
- **数据存储与历史记录**：将采集到的数据保存到数据库，支持历史查询。
- **告警通知**：当环境数据超过设定阈值（如土壤过干 / 过湿 /温度异常）时触发告警。
- **可视化展示**：仪表盘 (dashboard)，图表 (折线图 /柱状图) 呈现环境趋势。
- **设备控制**（可选）：控制泵、阀门、风扇等硬件设备，实现自动灌溉 /通风。
- **用户管理**（可选）：不同角色 (管理员 / 普通用户) 的权限管理。

## 四、安装与部署指导

（下面是一个通用示例，具体根据你项目中的 `esp32/`, `front/` 等文件夹结构补充）

1. **克隆项目**

   ```bash
   git clone https://github.com/DreamDonghao/smart_agriculture.git
   cd smart_agriculture
   ```

2. **后端部分**

   - 进入后端目录 (假设是 `backend/` 或 `esp32/`，根据你的结构改)：

     ```bash
     cd esp32
     ```

   - **构建**：根据你的构建系统 (CMake, Makefile 等) 执行构建命令。

   - **配置**：修改配置文件 (如 `config.ini` / `yaml` / `.env`) 来设置数据库地址、传感器参数等。

   - **运行**：启动后端服务，监听传感器数据或 API 请求。

3. **前端部分**

   - 进入前端目录 (假设是 `front/`)：

     ```bash
     cd front
     ```

   - 安装依赖：

     ```bash
     npm install
     ```

   - 启动开发服务器：

     ```bash
     npm run serve
     ```

   - 构建生产版本 (可选)：

     ```bash
     npm run build
     ```

4. **数据库**

   - 如果使用数据库 (如 SQLite / MySQL / PostgreSQL)：建立数据库，执行初始化脚本 (如果有)。
   - 配置后端连接数据库时的连接字符串。

## 五、使用示例

（这里你可以写几个典型场景 / 用例：监控一个温室、模拟传感器数据、触发告警等）

  1. 部署传感器（温度、湿度、光照）模拟器或真实传感器。
  2. 后端接受实时数据并写入数据库。
  3. 前端展示温度 / 湿度变化曲线。

- **示例 2 — 自动灌溉 (假设已有设备控制)**：
  1. 当土壤湿度低于设置阈值时，后端发控制指令给泵 /阀门设备。
  2. 泵打开灌溉，传感器继续上报湿度。
      3.湿度恢复到安全范围后，关闭泵。

## 六、项目扩展与未来计划

这里是一些可以考虑但目前可能还没实现 / 可以未来添加的特性 (喵喵，我猜你会想加)：

- **机器学习 /预测模块**：利用历史环境数据，对未来温湿度趋势或灌溉需求进行预测。
